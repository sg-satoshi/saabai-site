# Site Factory — Prompt Simplification Design

**Date:** 2026-05-27
**Status:** Approved for implementation

## Problem

Recent sites generated by Site Factory are noticeably lower quality than earlier ones — basic-looking navs, poor layouts, weak image selection. The root cause is over-specification: the current prompt system has grown to 245+ lines of prescriptive rules per generation call. Claude spends its attention budget on rule compliance rather than design craft.

Reference sites built with the older, simpler prompt (Heaven Thai Massage, Tributum Law) are significantly better. Those were generated when themes didn't exist and Claude had creative latitude.

## Principle

Give Claude a creative brief, not a technical specification. The prompt tells Claude *what kind of site to make*, not *how to make every pixel of it*.

## Scope

Three files change. Nothing else:
- `app/api/site-factory/generate/route.ts`
- `app/api/site-factory/generate-multi/route.ts`
- `lib/site-themes.ts`

Edit route, versioning, publish/draft system, and admin UI are untouched.

## Design

### 1. System Prompt

Single responsibility: set the quality bar, state hard output rules, define what "great" means. No layout dictation, no pixel specs.

```
You are an elite web designer who creates stunning, conversion-optimised websites
for Australian small businesses. Your output rivals Lovable, Webflow, and Framer —
not generic templates. Every site feels custom-designed and worth $5,000+.

HARD RULES:
- Output ONLY raw HTML starting with <!DOCTYPE html>. No markdown, no code fences.
- ALL CSS in one <style> tag. ALL JS in one <script> tag before </body>.
- No external CSS frameworks. Pure CSS Grid and Flexbox.
- Mobile-first. Breakpoints: 768px tablet, 1024px desktop.
- All images: loading="lazy", explicit width/height, real Unsplash photo IDs that
  genuinely match the business niche and theme aesthetic.
- No em dashes (—). Use commas, colons, or rewrite.
- Sticky nav: transparent to opaque on scroll. Hamburger menu on mobile.
- IntersectionObserver reveals (.reveal class). Count-up stats when in view.
- Full SEO: JSON-LD LocalBusiness (most specific @type), Open Graph, Twitter Card,
  semantic HTML5.
```

Target: ~15 lines. Current: 40+ lines in the system prompt alone.

### 2. Theme Structure (`lib/site-themes.ts`)

Each theme definition reduces from ~30 prescriptive rules to 8-9 lines of creative direction:

```typescript
interface ThemeDef {
  aesthetic: string;    // 1-sentence personality description
  palette: string;      // CSS custom property definitions
  fonts: string;        // Google Fonts import + heading/body spec
  hero: string;         // 1-sentence hero mood and layout
  rules: string[];      // 2-4 hard visual rules (non-negotiable character traits)
  dark: boolean;
}
```

The `rules` array is for character-defining constraints only — the things that make a theme unmistakably itself. Not pixel values, not section ordering, not exact border-radius on every element.

**Example — PRESTIGE:**
```typescript
prestige: {
  dark: false,
  aesthetic: "Cinematic legal authority. Deep navy, forest teal, cold whites. Measured and trustworthy.",
  palette: "--primary:#0D1D2B;--secondary:#006B5E;--accent:#2AA896;--bg:#F7FAFC;--surface:#FFFFFF;--text:#0D1D2B;--text-muted:#3D4946",
  fonts: "Inter all weights. Headings: 700, -0.02em tracking. Body: 400, 1.75 line-height.",
  hero: "Full-bleed photo with deep navy overlay (rgba 0.62), left-aligned content block, small teal pill eyebrow label above headline.",
  rules: [
    "Every section heading preceded by a small pill eyebrow label (teal border + text on light sections, aqua on dark)",
    "Section backgrounds alternate strictly: white and #F1F4F6 — never two identical in a row",
    "Stats band: deep navy background, teal numerals, white labels",
  ],
}
```

**Example — APEX:**
```typescript
apex: {
  dark: true,
  aesthetic: "Industrial bold. Dark navy, vivid orange, zero softness. Built for trades.",
  palette: "--primary:#0c1a30;--secondary:#f97316;--accent:#ea580c;--bg:#0c1a30;--surface:#162035;--text:#f1f5f9;--text-muted:#94a3b8",
  fonts: "Montserrat 900 uppercase headings, Roboto 400 body.",
  hero: "Diagonal cut layout with orange geometric shape right side, hard left-aligned content, 4px orange rule above headline.",
  rules: [
    "border-radius: 4px everywhere — no soft corners",
    "Full dark site: all section backgrounds #0c1a30 or #162035 only",
    "Orange (#f97316) reserved for CTAs, accent lines, and stat numbers only",
  ],
}
```

All 9 themes follow this pattern. The `ThemeDef` interface in `site-themes.ts` updates to match the new shape. Fields removed: `sections`, `headingCss`, `bodyCss`, `fontImport`. Fields added: `aesthetic`, `fonts`, `hero` (renamed/consolidated). Fields kept: `palette`, `rules`, `dark`. Any references to the removed fields in the generate routes are replaced with the new fields during implementation.

### 3. User Prompt (single-page)

Replace the 120-line `━━━` structure with a clean brief:

```
Build a complete production website. Write real, compelling copy — not Lorem Ipsum.

BUSINESS: {name} | NICHE: {niche} | LOCATION: {location}
PHONE: {phone} | EMAIL: {email} | SERVICES: {services}
{description block if provided}

THEME: {themeKey}
{aesthetic}
{palette}
{fonts}
{hero}
{rules as numbered list}

SECTIONS (in this order):
Nav → Hero → Trust bar (4 signals with SVG icons) → Services (6 cards, CSS Grid 3/2/1 col) →
Process (3 numbered steps) → Stats (4 count-up metrics, data-target attribute) →
Testimonials (3 cards, Australian suburbs, 5 stars, specific quotes) →
About (photo left + story right) → FAQ (6 questions, accordion with JS toggle) →
CTA band → Contact (details + form POSTing to https://www.saabai.ai/api/site-factory/lead
with {name,email,phone,message,siteSlug:"{slug}"}, success/error state) → Footer

Nav must include a working hamburger menu on mobile.
Pick Unsplash photo IDs that genuinely look like this business in this location.
```

Target: ~25 lines. Current: 120+ lines.

### 4. Multi-Page Generation (`generate-multi/route.ts`)

Same philosophy applied per page. Each page call gets:
- The same theme brief (palette, fonts, hero, rules)
- A page-specific content brief (1-3 sentences on what this page achieves)
- A shared design DNA block (CSS variables + font imports, consistent across all pages)

Page-specific briefs:
- **Home:** Full site as above
- **About:** Firm story, founding, team bios, values — no hero photo, use a warm interior/team photo
- **Services / Practice Areas:** One section per service, detailed descriptions, no hero
- **Contact:** Full contact details, map embed placeholder, form — minimal layout

The multi-page route generates pages sequentially. The "design DNA block" is a literal string of `:root { --primary:...; --secondary:...; }` CSS variables plus the Google Fonts import, constructed from the theme's `palette` and `fonts` fields and included verbatim in every page's prompt. No attempt to extract CSS from page 1 — keep it simple, let the theme brief do the consistency work.

## What Does Not Change

- `lib/site-versions.ts` — versioning untouched
- `app/api/site-factory/edit/route.ts` — edit route untouched
- `app/saabai-admin/site-factory/SiteFactoryClient.tsx` — admin UI untouched
- Theme names and slugs (onyx, coast, edge, grove, slate, spark, craft, prestige, apex) — preserved so existing sites are unaffected
- `NICHE_THEME_DEFAULTS` mapping — preserved

---

## Success Criteria

- Generated nav has a proper hamburger menu with working JS toggle
- Hero image is a real, relevant Unsplash photo (not a placeholder ID)
- Visual quality matches or exceeds Heaven Thai Massage and Tributum Law
- Total prompt length (system + user) under 50 lines
- Build passes: `npm run build` clean after changes
- Both single-page and multi-page generation work without errors
